# Workflows Under Review

This folder stores workflow agents/skills that are still being evaluated before entering the main catalog.

Difference from specialist agents:
- Specialist agents own a domain area, such as frontend, QA, product, or sales.
- Workflows guide how work is conducted, such as TDD, feedback review, final verification, or isolation with worktrees.
- These files do not define global hard gates. They describe recommended protocols that an orchestrator, pipeline, or main agent can invoke when appropriate.

Items in this folder:
- `browser-testing-with-devtools.md` — UI verification in a real browser with DevTools, console, DOM, network, and performance.
- `code-simplification.md` — code simplification while preserving behavior.
- `context-engineering.md` — context curation for sessions and agents.
- `deprecation-and-migration.md` — safe system discontinuation, migration, and removal.
- `doubt-driven-development.md` — optional adversarial review for high-risk decisions.
- `incremental-implementation.md` — implementation in small and verifiable slices.
- `source-driven-development.md` — development guided by official sources and current documentation.
- `test-driven-development.md` — test-driven development discipline.
- `receiving-code-review.md` — technical triage and response to review feedback.
- `verification-before-completion.md` — verification checklist before communicating completion.
- `using-git-worktrees.md` — evaluation and safe use of worktrees for work isolation.
