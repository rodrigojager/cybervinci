# @cybervinci/csharp-kit

CyberVinci/Theia extension for C# and .NET workflows.

It contributes:

- a C# Kit panel with Solution Explorer-style solution/project/file inventory;
- project creation for common `dotnet new` templates and `.sln`/`.slnx` solution membership;
- solution-level restore/build/clean/rebuild/format/test commands for `.sln` and `.slnx` workspaces, plus MSBuild restore/build/clean/rebuild tasks for `.slnf` solution filters;
- workspace inspection for `.sln`, `.slnx`, `.slnf`, `.csproj`, `.cshtml`, `.razor`, Razor import files, `.pubxml` publish profiles, workspace `global.json` SDK and `dotnet test` runner selection, `.editorconfig`/`.globalconfig` C#/dotnet/analyzer settings, workspace `dotnet-tools.json` local tools, generated `dotnet tool restore` and `csharp-ls` local-tool install tasks, `.runsettings` test settings, nearest `Directory.Packages.props`, inherited `Directory.Build.props`/`Directory.Build.targets` and workspace `NuGet.config` files, including missing solution project entries;
- workspace-local `.cybervinci/csharp-kit.json` health reporting for decoupled Roslyn, CoreCLR and language-server adapter configuration;
- JSON schema registration for `.cybervinci/csharp-kit.json`, including validation/default snippets for workspace-local Roslyn, CoreCLR and C#/Razor language-server adapter settings;
- `C# Kit: Open Adapter Config` command and panel action to create/open the non-overwriting adapter config template without requiring a C# project;
- Roslyn sidecar readiness status for `@cybervinci/memory-roslyn`, configured through env vars, workspace-local `.cybervinci/csharp-kit.json` or the installed package layout;

This package is installed through the `Modificacoes/CSharp Kit` feature folder.

After `npm run compile`, run `npm run test:csharp-ls-e2e` to perform the opt-in real adapter check. It creates a temporary workspace, installs `csharp-ls` as a local dotnet tool, runs the C# Kit LSP initialize probe and removes the temporary workspace by default.

Run `npm run test:dotnet-workflow-e2e` for the opt-in real .NET workflow check. It creates a temporary solution and console project through `CSharpKitServiceImpl`, inspects the workspace, writes launch/tasks/config files, runs C# Kit build diagnostics and removes the temporary workspace by default.

Run `npm run check:runtime-readiness -- --workspace <path>` after compiling to inspect a real workspace and print the C# Kit runtime/adapters readiness summary. Use `npm run check:runtime-readiness:full -- --workspace <path>` as a strict gate: it exits non-zero until dotnet, CoreCLR debug, C# LSP and Razor LSP adapters are discoverable and both language servers pass an LSP initialize probe.
